Online Shop

This web application is a online shop example I have built to showcase my back end user interface skills.
This site example is based on a 1 piece inventory per item, in theory all items are unique. Please note I used a starting template from HTML UP designs, this was done to help build a responvie layout and to import a CSS main styles files.
I have customized the template by diving into the CSS styles, changing colors and tweaking the layout. I have also built complete html pages that I required to fully built this online shop example.

This web application is composed of six main navigation options which are:

  • Home
  • Finds
  • Cart
  • Register
  • Log In
  • Log Out


The Finds section refers to the "Product List" section of the online shop example.
Throught he use of database - SQLAlchemy (Filebased for this example), the items are listed with items which have been sold being listed at the bottom.
The "SOLD" image is also automatically transpose onto the image once its bought. (Logic: Quantity=0).


Several modules where imported into this project, one of them being Flask_Login which was leveraged for our authorization needs. As you can see below, the login page works as inteneded. An incorrect username was entered and a message was delivered to the user, in the second example below.


To access the Cart section, a valid user is required to login. If the "Cart" button is clicked prior to login in, the user will be redirected to an "Unauthorized" message.
I built logic behind the cart to exclude duplicates and only list each item once. It also adds up the summation and displays the total cost.
The checkout button is just a "dummy" place holder at this moment, ready to interact with "Stripe" a well known financial services and software company. "Stripe" can be leverage for all the payment transaction fees.
As this is just an example I have not connected this web application to "Stripe".


Adding a product is only available to the Administrator of the website, updating and deleting an image to a post is also locked and accesible only to the Administrators.
To be redirected to the add section the adminastrator must login and manuall enter "/add" to the local url.
Once redirected the user sees a form which was built to add a new item and submit.


After the product is submitted, the user is redirected to the finds sections where the new item has been added to the item list.
At this point the user can update the items image. The second post below shows me using the delete function to delete the "New Shirt" item.

Repository is set up as usual, html code are found in template, styles/etc. are found in static & main serrver is found on the main folder.


Flask was heavly used in the construction of this project, module variants of it as well:

  • Flask - is a micro framework written in python. Helps develop web applications.
  • Flask_sqlalchemy - is an extension of flask that adds support to SQLAlchemy
  • Flask_wtf - is an extension of flask that adds support to WT Forms. Provides interactive user interface for the user.
  • WTforms & WTforms-validators - hand in hand with the the module above we also imported the WT Forms to help build our interactive forms.
  • Flask_bootstrap - is an extension of flask that packages Bootstrap.
  • Flask_login - is an extension that provides user session management for Flask
Werkzeug.Security was imported into the project, which is a python library that is used for implementation of web application gateway interface (WSGI) applications.
We leveraged it to handle our user registration needs.
Also imported was Functools,Django, Collection & OS.

First section of the code is the main server set up, database connection & login manager setup.


The Database created handles 3 tables: Inventory, User and Cart.



Functions:
admin_only(f) - wrap function which set admin only restricition. home() - indexes home website, also pulls all product into a query and sorts it by quantity descending order. Passes query to Index home page.
login() - handles user login and user messages if errors are encountered.


register() - registers a new user, checks for unique email and converts password to hash and saves the hash. No passwords are saved.


add_product() - uses the AddProductForm created and allows user to enter a new product. Adminastrator only level. delete() - deletes a specific item, item no. is carried accross to allow the indexing and deletion in the database. uploader() - uploads image per specific item, item no. is carried accross to allow indexing and appending in the database. logout()- simply logs out the user.


cart() - builds cart for view per user login, also sorts out all duplicates.
additems() - allows user to add selected item into their cart.
removeitems() - allows user to remove selected item from their cart.
